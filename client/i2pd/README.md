Это клиентская часть модифицированного i2pd роутера (https://github.com/PurpleI2P/i2pd). Патч убирает лимит в 2500 известных роутеров, так же устанавливает начальное окно в 512 байт вместо 10 в оригинальной реализации, это необходимо для повышения производительности. Папка `data` монтируется в `docker container`, поэтому на ней должны быть установлены правильные привелегии `sudo chown -R 1000:1000 ./data`. Файл `data/i2pd.conf` отвечает за настройку роутера, `data/tunnels.conf` за тунелли. Вам не стоит вручную создавать тунели, лучше используйте скрипт `python3 gen.py > data/tunnels.conf` для генерации тунелей. Замените `[CENSORED].b32.i2p` на адрес вашего роутера, он должен выглядеть как то так: `tr26d3g4hviaqufmld6r16ikpdvjvd6g5qwdemnbuffck2fh663a.b32.i2p`

# Первый запуск
- Соберите контейнер командой `./scripts/build.sh`
- Настройте клиентский конфиг `data/i2pd.conf` как вам удобно (опционально)
- Замените `[CENSORED].b32.i2p` в файле `gen.py` на адрес вашего сервера
- Сгенерируйте конфиг для тунелей `python3 gen.py > data/tunnels.conf`
- Установите правильные привелегии на папку `data`, через `sudo chown -R 1000:1000 ./data`
- Запустите роутер, командой `./scripts/run.sh` 
- Откройте веб браузер для проверки того что все работает http://127.0.0.1:7070

# Папка `scripts`
- Старт через `./scripts/run.sh`
- Выключение `./scripts/stop.sh`
- Экспорт docker образа через `./scripts/export.sh`, эта команда создаст файл `i2pd_image.tar`
- Импорт образа через `./scripts/import.sh image_name.tar`
